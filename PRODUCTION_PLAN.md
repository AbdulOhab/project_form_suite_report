## Context

প্রোজেক্ট ৮০% শেষ। বাস্তবে অডিট করে দেখা গেছে: **server ~40-50% production-ready, client ~70-75% production-ready**। কোর ফিচার (auth, notice CRUD, answer submission, user management, data aggregation) কাজ করে — কিন্তু production-এ দেওয়ার আগে security, config, এবং কিছু stub/broken জায়গা ঠিক করতে হবে।

### Phase 2: Auth Middleware on All Protected Routes (3-4 hours)

**Goal:** Every endpoint requires a valid JWT, except `/`, `/submit`.

1. **Create `server-side/middleware/requireRole.js`:**
   ```js
   module.exports = (...allowedRoles) => (req, res, next) => {
     if (!allowedRoles.includes(req.userData?.userRole))
       return res.status(403).json({ error: "Forbidden" });
     next();
   };
   ```

2. **Apply `authMiddleware` to every route file:**
   - `noticeRoutes.js` — all CRUD requires auth
   - `answerRoutes.js` — all requires auth
   - `dataCheckRoutes.js` — all requires auth + `requireRole` (admin/zonal/branch/thana per path)
   - `sumsDataRoutes.js` — admin-only via `requireRole("admin")`
   - `usersCreateUpdateRoutes.js` — admin-only
   - `dashboardRoutes.js` — auth required; `/thana-users`, `/zonal-users`, `/branch-users` need role check

3. **Fix logout endpoint** [server-side/controller/authController.js:165-169](server-side/controller/authController.js#L165):
   ```js
   exports.logout = (req, res) => res.json({ ok: true });
   ```
   (Client clears localStorage; no server-side state to clear unless using a token blocklist.)

**Verify:**
- `curl http://localhost:5053/all-notice` returns 401.
- With valid token: returns data.
- Branch user calling `/admin/data-checkout/...` returns 403.

### Phase 3: Hardening Middleware (1-2 hours)

   const helmet = require("helmet");

### Phase 9: VPS Deploy via Docker (2-4 hours)

1. **VPS prep** (Ubuntu/Debian assumed):
   ```bash
   apt update && apt install -y docker.io docker-compose-plugin git
   ufw allow 22 && ufw allow 80 && ufw allow 443 && ufw enable
   ```
2. **Clone repo** to `/opt/project-form-suite`; create `.env` files on the server (don't commit prod secrets).
3. **Add Caddy as reverse proxy** in `docker-compose.yml`:
   ```yaml
   caddy:
     image: caddy:2-alpine
     ports: ["80:80", "443:443"]
     volumes:
       - ./Caddyfile:/etc/caddy/Caddyfile
       - caddy_data:/data
   ```
   `Caddyfile`:
   ```
   instancereport.deepseahost.com {
     reverse_proxy client:80
   }
   instancereportbackend.deepseahost.com {
     reverse_proxy server:5053
   }
   ```
   Caddy handles Let's Encrypt automatically.
4. **Enable MongoDB auth** — add `MONGO_INITDB_ROOT_USERNAME/PASSWORD` to `mongodb` service; update `MONGODB_URI` to include creds.
5. **Point DNS** for both domains at VPS IP.
6. **First boot:**
   ```bash
   docker compose up -d --build
   docker compose exec server node seeder.js  # one-time
   ```
7. **Verify** logs: `docker compose logs -f server client caddy`.

**Verify:** Login from production URL works, full smoke test (Phase 7) runs green on live.
